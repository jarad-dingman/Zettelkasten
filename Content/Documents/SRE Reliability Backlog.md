**Context:** 
	These are the tasks that help make up the SRE stages for integrating with teams.

## Summary

- The 20 subtasks of SRE work.

| Title                                                                                         |
| --------------------------------------------------------------------------------------------- |
| Identify current pain points (Incidents, MIs, Team interviews, Wikis & Tribal Knowledge)      |
| Assess Reliability Maturity Level (0-5 scale)                                                 |
| Build or augment score card with key reliability metrics                                      |
| Define target metrics – Service level Objectives (SLOs)                                       |
| Define reliability backlog and align with product teams on reliability backlog prioritization |
| Define Error Budgets to balance “feature vs stability”                                        |
| Provide PROD equivalent non-prod environments for testing                                     |
| Implement full stack monitoring                                                               |
| Implement full stack observability                                                            |
| Automate tests, releases and operations                                                       |
| Implement performance testing                                                                 |
| Implement failover testing (Chaos Engineering)                                                |
| Implement safe release process (Feature toggles/Canarying/Blue-Green deployments)             |
| Design for reliability (Reliability Patterns) and review designs                              |
| Review major incidents and conduct blameless postmortems to identify root cause               |
| Establish production readiness review template and conduct reviews (every release)            |
| Ensure runbooks are documented and uptodate                                                   |
| Ensure playbooks are documented and uptodate                                                  |
| Ensure that all incidents have documented RCAs and solutions for prevention                   |
| Address gaps in release management, incident management and SLOs                              |
